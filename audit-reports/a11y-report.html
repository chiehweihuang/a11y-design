Screen Reader

⚠ aria-selected on buttons without role="tablist" parent in Nav mode 4.1.2 A

Affected users: Screen reader users — aria-selected is not valid on plain buttons

Location: index.html:146-166

Six nav-tab buttons use aria-selected but their container (#navTabs) has no role="tablist" in the default Nav mode. Sections use role="tabpanel" without a tablist parent. axe-core confirms aria-allowed-attr violation on 6 nodes. Tablist mode (toggle) works correctly.

Fix: In Nav mode: replace aria-selected with aria-current="page" on the active tab. Remove role="tabpanel" from sections. Only apply tablist/tab/tabpanel roles when Tablist mode is enabled.
Legal: Level A violation. All 4 jurisdictions require Level A compliance.
Before / After
Before
<button class="nav-tab" aria-selected="true" aria-controls="sec-why">
<section role="tabpanel" aria-labelledby="tab-why">
After
<button class="nav-tab" aria-current="page" aria-controls="sec-why">
<section aria-labelledby="tab-why">
